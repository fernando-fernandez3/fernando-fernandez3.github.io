Getting AI Agent Notifications on Telegram (Setup Guide)

Fernando Fernandez 9 min read 1,751 words
By Fernando Fernandez · runs 10+ AI cron jobs daily, all delivering to Telegram

Most AI agent output lives in a terminal you never look at.

The agent ran at 6am. It did its thing. The output is sitting in a log file. You found out when you happened to open a terminal three hours later, or you didn't find out at all.

If nobody sees the output, the automation isn't doing much.

Getting your agent to reach you where you actually are (your phone) is what makes the whole system feel real. This post covers how to set that up with Telegram: why Telegram specifically, how to create the bot, how to connect it to your agent, and some patterns that are genuinely useful once you have it running.

Why Telegram

A few reasons, none of them complicated.

It's free. The API is free, the app is free, the bots are free. You're not paying $50/month for a Slack workspace to deliver notifications to yourself.

The API is good. Clear documentation, simple HTTP interface, well-maintained. I've used it for a couple years without it randomly changing or breaking.

It has topics. Within a single chat, you can create separate threads for different subjects. I have topics for Spanish practice, work stuff, family coordination, and running. My agent delivers to the right topic based on what the job is. One chat, organized. Not a flood of unrelated messages in a single stream.

Group chats work well. I added my wife to a family group with me and my agent. That turned out to be more useful than I expected. More on that in a minute.

The alternative people usually ask about: Discord. It works fine. The bot setup is similar. I use Telegram because it feels more like messaging a friend than like a Discord server, and that matters when you're receiving multiple agent notifications a day. Discord started feeling noisy. Telegram didn't.

Email is the other answer people give. I actively avoided email because I get too much of it and I'd start ignoring agent notifications the way I ignore newsletters. Telegram gets a different kind of attention from me.

Setting Up the Bot (30 Minutes)

Everything runs through a Telegram bot. Your agent doesn't message you directly: it messages via a bot you control.

Step 1: Create the bot

Open Telegram and search for @BotFather. That's Telegram's official bot that creates other bots. Start a chat with it and send /newbot.

BotFather asks for a name (what people see in the chat header) and a username (must end in bot). Pick whatever. For a personal agent setup, something like MyAgentBot or FernandoAgentBot is fine. The name doesn't matter much. The username is how you'll reference it.

Once created, BotFather gives you a token. It looks like: 7234567890:AAFhKJG4lmn8opqr_stvwxy1z2_ABCDE. Copy it. This is the only credential your agent needs to send messages.

Step 2: Get your chat ID

Your agent needs to know where to send messages. The "where" is your chat ID, a number that uniquely identifies your conversation with the bot.

Start a chat with your new bot by searching for its username and sending it any message: "hello" is fine.

Then open this URL in your browser, replacing YOUR_TOKEN with the token from step 1:

https://api.telegram.org/botYOUR_TOKEN/getUpdates

You'll see a JSON response. Look for "chat": {"id": 123456789}. That number is your chat ID. It might be positive (personal chat) or negative (group chat). Write it down.

Step 3: Add to your environment

In your .env file:

TELEGRAM_BOT_TOKEN=7234567890:AAFhKJG4lmn8opqr_stvwxy1z2_ABCDE
TELEGRAM_CHAT_ID=123456789

That's the whole setup. Test it immediately:

import httpx
import os

token = os.environ["TELEGRAM_BOT_TOKEN"]
chat_id = os.environ["TELEGRAM_CHAT_ID"]

httpx.post(
    f"https://api.telegram.org/bot{token}/sendMessage",
    json={"chat_id": chat_id, "text": "It works."}
)

Run that. If your phone buzzes with "It works.": you're done with setup.

Connecting Your Agent

Once the bot is working, there are two ways to use it in your agent setup.

Option 1: The agent calls the Telegram API directly

Give your agent the bot token and chat ID as environment variables, include the send function in its toolkit, and let it decide when to message. This is what I do for most cron jobs. The agent runs, produces output, sends it to Telegram. Done.

The simplest send function:

import httpx
import os

def send_telegram(text: str, topic_id: int | None = None) -> None:
    token = os.environ["TELEGRAM_BOT_TOKEN"]
    chat_id = os.environ["TELEGRAM_CHAT_ID"]

    payload = {"chat_id": chat_id, "text": text, "parse_mode": "Markdown"}
    if topic_id:
        payload["message_thread_id"] = topic_id

    httpx.post(
        f"https://api.telegram.org/bot{token}/sendMessage",
        json=payload,
        timeout=10
    )

The topic_id parameter is how you route to specific topics in a group chat. Each topic has a numeric ID you can get from the URL when you're in the topic in the desktop app.

Option 2: OpenClaw or your agent runtime handles delivery

If you're using OpenClaw, Telegram is a configured channel. Your agent just replies normally and OpenClaw routes it to your phone. No extra code. This is the simpler path if your runtime supports it.

Using Topics to Stay Organized

This is the part people skip and then regret once they have a few cron jobs running.

Without topics, your Telegram chat with your agent looks like a firehose. Morning briefing, Spanish lessons, health check alerts, memory maintenance confirmations, job failure notifications: all in one stream. You'll start ignoring things because it's too noisy.

Topics fix this. Each topic is a separate thread in the same chat. My setup:

  • Briefings: morning news digest, daily summary
  • Spanish: language practice lessons and quizzes, three per day
  • Work: JIRA digest, meeting prep notes
  • Fitness: training log updates, race reminders
  • System: health check alerts, cron failure notifications
  • Family: family coordination (this is a group with my wife)

Your cron jobs need to know which topic they're sending to. Each topic has an ID (a number). Get it from the desktop app: open the topic, look at the URL, it'll have ?thread=12345 or similar at the end. That number is your message_thread_id.

Hard to give exact steps because Telegram occasionally changes the desktop UI. The approach that always works: send a message in the topic via the app, then call getUpdates again. The response will include the message_thread_id for that message.

Once you have the IDs, add them to your env:

TELEGRAM_TOPIC_BRIEFINGS=12345
TELEGRAM_TOPIC_SPANISH=12346
TELEGRAM_TOPIC_SYSTEM=12347

Your send function loads the right one based on which job is running.

The Group Chat Setup

This is worth its own section because it changed how useful my agent is to my actual life, not just my work life.

I added my wife to a Telegram group, then added my agent bot to the same group. The bot delivers family coordination messages to both of us.

Before this: I'd have a mental note about something family-related. The camping trip this weekend, we're taking the truck because we need the trailer hitch. Mental note. Which I would forget to communicate. Which would cause a problem.

Now: the agent runs a family coordination check three days before any scheduled event. It sends a reminder to the group. My wife sees it the same time I do. The information gets to both of us without me having to remember to pass it along.

The other thing it does: when we're planning something: a road trip, home project, scheduling something with the kids: I can tell the agent in the family group chat and it picks that up for context. It's not magic. It's just that the information is in a place the agent can see and act on.

To add a bot to a group: open the group, go to Add Members, search for your bot's username. Done. The bot will have access to messages in the group.

One thing to configure: by default in Telegram, bots can only read messages if they're directly mentioned or if the group has been set to allow bots to read all messages. Go to the group settings and check the Privacy settings for bots. For a group where it's just you and your wife and a bot you control, this isn't a concern.

Markdown in Messages

Your agent's output will be more readable if you format it. Telegram supports a subset of Markdown when you include "parse_mode": "Markdown" in the request.

What works:
- *bold* → bold
- _italic_ → italic
- `inline code` → inline code
- ```code blocks``` → code blocks

What to watch out for: if your text includes underscores or asterisks in normal content (variable names, file paths, anything technical), they'll try to parse as markdown and break the formatting. Either strip them or use parse_mode: "MarkdownV2" which requires escaping but is more predictable.

For agent-generated summaries, I use plain Markdown: *Section Header* for headers, bullet points for lists, \inline`` for any technical terms. Readable without being fancy.

Debugging When It Doesn't Work

Three common issues:

Bot not receiving messages / getUpdates returns empty
Your bot has never received a message. Send it something from the Telegram app first, then call getUpdates again.

403 Forbidden
Your chat ID is wrong, or the user/group has blocked the bot. For personal chats: message the bot first. For groups: make sure the bot is in the group.

Message sends but appears in wrong chat
You have multiple Telegram accounts or the chat ID in your env is stale. Re-run the getUpdates step to confirm the current chat ID.
