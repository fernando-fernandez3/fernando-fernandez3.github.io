Google's Free Agent IDE Supports Claude. The Manager Surface Is What Interests Me.

Fernando Fernandez 7 min read 1,310 words
Google Antigravity hit public preview on February 27. It's free, supports Claude Sonnet 4.5, and has async agent orchestration. The gap between it and headless cron is one specific thing.

Google dropped Antigravity into public preview on February 27. Free for individuals, cross-platform, VS Code fork. The coverage focused on model optionality: you can run Gemini 3 Pro, Claude Sonnet 4.5, or GPT-OSS without paying for API access.

What I hadn't seen anyone write about specifically: the Manager Surface. That's the part that's actually new for people building background agent systems.

What Antigravity Actually Is

Two interfaces in one tool. The Editor View is a standard AI-powered coding environment: tab completions, inline commands, the things you'd expect from a Cursor competitor. That part isn't interesting to me.

The Manager Surface is different. It's a dedicated interface where you spawn agents, watch them work, and orchestrate multiple agents running asynchronously across different workspaces. You delegate a task. The agent plans it, executes it across your editor, terminal, and browser without you touching anything, and comes back with artifacts: screenshots, walkthroughs, a summary of what it did. You review. You move on.

Multiple agents can run in parallel. You're not waiting for one to finish before the next starts. You're the manager, not the operator.

This is the closest thing Google has shipped to what I've been running manually for the past year.

What I've Been Running

My setup has 10 agents on cron. Cron is a Linux scheduler — you tell it "run this at 6am every day" and it does. No IDE required. Nothing needs to be open. The machine is doing its thing whether I'm awake or not.

At 6am, my morning briefing agent fires. It pulls my calendar for the day, checks unread email, reads my Telegram messages from overnight, checks weather, and writes a 4-paragraph summary to a markdown file. When I wake up and open the file, it's done. I never asked. I never typed a prompt. I didn't even know it was running.

The other agents: memory consolidation at 6:15am, health check at 9am, email triage at 7am, pool chemistry monitoring when sensors post new readings. None of those are coding tasks. None of them touch GitHub. They're just things that need to happen and don't need my hands on them.

Total cost: about $40 a month. All 10 agents, every day.

The Claude Sonnet 4.5 Detail

Antigravity supports Claude natively. Not a plugin. Anthropic is a listed model provider alongside Gemini. The public preview gives you Gemini 3 Pro with generous rate limits for free. Claude Sonnet 4.5 is available when you want it, which presumably means paying for API access or having your own key.

For anyone in the Claude API ecosystem, this is significant. Google built an agent-first IDE where your existing Claude setup works. You don't have to choose between Google's tooling and Anthropic's models.

If you've been building Claude-based agents and want a more structured environment to manage them visually, Antigravity is now a real option. That wasn't true six months ago.

Where Antigravity Is Better

Tasks that require UI interaction. My cron agents are scoped to API calls, file reads, and structured data. They don't click through a browser. If I need an agent to fill out a form, scrape a page that requires JavaScript rendering, or verify that a UI change looks right, my setup needs extra tooling. Antigravity handles all of that natively. The agent moves across editor, terminal, and browser as one continuous task. That's a real capability gap.

Visual output. When Antigravity finishes a task, it sends back screenshots and walkthroughs as artifacts. My agents write text to markdown files. That's fine for data-heavy output. It doesn't work when "did it work" requires looking at something rendered. For UI work, the artifact model is genuinely better.

Zero setup for model access. Antigravity gives you Gemini 3 Pro with no API key in the public preview. If you're just starting out, not having to create an Anthropic account, generate a key, and wire up billing is real friction removed. The on-ramp is faster.

Parallel agent management in a UI. Running multiple agents simultaneously in my setup requires multiple cron processes and coordination logic to avoid conflicts on shared files. Antigravity manages this visually. You can see what each agent is doing, pause them, or redirect them. For anyone who prefers visual control over terminal-level configuration, that's a better experience.

Where DIY Cron Still Wins

Truly headless operation. This is the gap that matters most. My agents run at 3am when my machine is quiet and nothing is open. Antigravity requires the application to be running. If you want agents that work whether or not you're at your desk with the IDE open, you still need to build the cron layer yourself. That's not a knock on Antigravity — it's built around developers interacting with it during the day. But for automation that needs to fire on a schedule regardless of what you're doing, headless wins.

Predictable costs. Antigravity is free in public preview. That's a growth strategy, not a pricing model. I don't know what GA pricing looks like. My direct API costs are controlled: Haiku for health checks and quick scans, Sonnet for anything requiring judgment, Opus very sparingly. I pay for what I use, and the bill is predictable. That matters when you're running 10 agents daily with real compute behind each one.

Tasks outside the dev stack. Antigravity's architecture is built around software development workflows. Editor, terminal, browser for testing. My pool chemistry agent doesn't have a codebase. My family calendar briefing doesn't open PRs. If your background agents handle life outside code, you're going custom regardless of what IDE you're using.

Model routing by cost and task type. I route tasks to different models based on complexity and cost. Simple checks go to Haiku at fractions of a cent. Judgment calls go to Sonnet. I can drop in a local Llama model for zero-cost tasks when the quality bar is low. Antigravity's model choices are Antigravity's model choices — you pick from what they support.

Who Should Use Which

Antigravity makes sense if you're a developer who wants to offload complex, UI-touching tasks during the day. Delegating a feature implementation, testing a UI change, running a research task that requires browsing multiple sites. The Manager Surface handles parallel work cleanly and the setup cost is low.

The DIY cron path makes sense if your agents fire on a schedule independent of whether you're at your desk, handle tasks outside the dev stack, or need predictable costs across high-frequency runs. The headless constraint is real and Antigravity doesn't close it.

The honest answer for a lot of developers is probably both. Antigravity for interactive daytime agent work. Cron for overnight automation that shouldn't require your machine to have an IDE open.

Google's Manager Surface is the closest thing to the multi-agent cron architecture in a packaged product. The headless gap is the specific thing that keeps me on the DIY path for background agents. If they close it, the calculation changes.
