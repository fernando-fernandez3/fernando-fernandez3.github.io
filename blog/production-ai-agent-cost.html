What Does It Actually Cost to Run a Production AI Agent? My $3/Day Setup

Fernando Fernandez 6 min read 1,023 words
By Fernando Fernandez · runs 10+ AI cron jobs daily, tracks token costs obsessively

The short answer: about $3 a day.

That covers 10 scheduled cron jobs running throughout the day, plus whatever interactive sessions I do manually. Morning briefing, memory maintenance, health checks, Spanish practice, JIRA review, family calendar sync, all of it.

Before I optimized the setup, it was closer to $7-8 for the same workload. The difference was model routing, which is covered in a separate post. This post is about the actual numbers and what drives them.

What I'm Running

Ten cron jobs on a Linux server (WSL2 on Windows with a 3090). All talk to the Anthropic API directly. Here's the breakdown by model:

Haiku jobs (~60% of my runs):
- Health checks (3x daily)
- Memory maintenance (nightly)
- Daily note extraction (nightly)
- Spanish vocabulary quiz (daily)
- Token usage report (nightly)

Sonnet jobs (~35% of my runs):
- Morning briefing (daily)
- Daily summary (nightly)
- JIRA ticket review (twice daily on weekdays)

Opus jobs (~5% of my runs):
- Used occasionally for complex interactive sessions, not scheduled cron jobs

The math on those percentages is that most of what runs is cheap. The jobs I care about reading (morning briefing, daily summary, ticket review) get Sonnet. Everything else gets Haiku.

My March 2026 Breakdown

These are my actual numbers for the first week of March:

Model Daily Avg Cost What It Covers
Haiku ~$0.40 5-6 jobs, health checks, maintenance
Sonnet ~$1.80 3 jobs, higher output quality
Opus ~$0.30 Occasional complex interactive sessions
Total ~$2.50 10+ cron jobs + interactive use

By month, that's around $75. For a personal AI setup that does things without me touching it, I think that's reasonable. One coffee a day, roughly.

Compared to what I'd spend on other productivity tools: Notion, Linear, various subscriptions that don't actually do work for me. But that comparison doesn't matter much. What matters is whether the output is worth $75/month. For me, clearly yes.

What Drives Costs Higher

A few things that made my bill higher before I fixed them:

Defaulting to Sonnet for everything. This is the most common mistake. When you're getting started, Sonnet feels safe. It's the "good" model. But Haiku is fine for health checks, file reads, compression, and templated tasks. Sonnet earns its cost for tasks where the output quality matters. For reads and checks, you're just paying more for the same result.

Not trimming context. Every file you load into a session is tokens. If your agent loads its full memory archive into every session, you're paying for all of it. I load MEMORY.md (the compressed summary, kept under 10KB) plus the relevant daily notes. The full archive is searchable on demand, not loaded by default.

Running expensive jobs too frequently. My JIRA review used to run every hour. Didn't need to. Twice a day is fine and costs half as much. Look at your schedule and ask: does this need to run this often?

Long sessions without thinking. Interactive sessions accumulate context as the conversation continues. If you're 40 exchanges deep in a planning session, each message is carrying the full history. Starting a new session for a new topic is cheaper and often just as good.

How to Set a Budget and Stay Under It

I run a nightly token report cron that writes a daily cost breakdown to a file. Once a week, I look at it.

The token report cron runs on Haiku and reads the API usage logs. It groups costs by job, shows the week-over-week trend, and flags if any job has gotten significantly more expensive (usually means context is growing unbounded somewhere).

The HTML dashboard I built reads those logs and visualizes them. Not required, but useful for the first few months while you're figuring out your pattern.

Budget target: I set a mental limit of $100/month. I've stayed well under it by maintaining the model routing habits. The nightly report makes it easy to see if anything is drifting.

If you want a harder limit, Anthropic has usage limits you can set in the API console. Set a monthly cap. If you hit it, jobs start failing (you'll get errors instead of responses). That's not ideal but it's better than surprise bills.

The Real Answer to "Can I Afford This?"

The question isn't whether $75-100/month is a lot of money. That depends on context.

The better question is: what does the agent actually do, and is that worth $75/month?

My setup handles my morning news digest, reviews my work queue twice a day, practices Spanish with me daily, maintains its own memory, monitors itself for failures, and coordinates family calendar stuff. If I hired a human assistant to do a tenth of that, it would cost more per hour than my monthly API bill.

But your setup might be smaller. If you run 2-3 cron jobs, your costs are proportionally lower. Haiku is cheap enough that you can run a health check and a morning briefing for under $0.50/day without trying.

Start small and add jobs that solve real problems. Watch the costs in the token report. Optimize when something looks wrong.
