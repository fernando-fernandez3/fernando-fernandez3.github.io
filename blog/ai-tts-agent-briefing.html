ElevenLabs vs Kokoro: Which TTS Actually Works for AI Agents?

Fernando Fernandez 8 min read 1,435 words
By Fernando Fernandez · runs 10+ AI cron jobs daily, including multilingual voice briefings

I've broken both.

ElevenLabs stopped me with rate limits in the middle of a Spanish lesson series. Kokoro wouldn't start because I'd missed a CUDA dependency. I've had cron jobs produce silent audio files, voice notes that played in Telegram as music instead of voice messages, and one very confused request from my wife asking why the agent was "speaking robot now."

This post is what I know after getting through all of that. If you're adding voice output to your AI agent and can't decide between ElevenLabs and a local TTS model, this is the actual comparison: not the one the product landing pages give you.

First, Do You Need Voice at All?

Before picking a TTS provider, the real question is whether voice is the right output format for what you're building.

I use voice output for two things: Spanish practice audio that I listen to on my phone during a run, and a morning briefing that plays while I'm doing something else. For everything else: JIRA summaries, task analysis, anything with data in it: I get a text message on Telegram.

Voice is worse than text when the output has structure. If your agent is summarizing five tasks with priorities and due dates, a text message is scannable in two seconds. A voice note forces you to listen through the whole thing to find what matters. I learned this the hard way. The briefing I converted to audio first ended up going back to text.

Voice is better when you want passive consumption. Running, cooking, driving. The agent delivers something and you absorb it without stopping what you're doing.

If your use case is passive consumption and the content doesn't need to be referenced later, voice is worth setting up. If not, skip TTS and spend the time on something else.

The Two Options

Two realistic choices for getting text-to-speech into an AI agent setup:

ElevenLabs: cloud API, high quality, free tier exists, costs money at scale. Works for any language. Has a voice library with native speakers for dozens of languages. The multilingual model is genuinely good at non-English.

Kokoro: local model you run yourself. Free forever, runs on a GPU you already own. English-first (other voices exist but English is where it shines). Requires setup, a real machine to run on, and enough VRAM to be practical.

There's no universal winner. The right answer depends on three things: what language you're generating, whether you have a GPU, and how many characters of audio you're generating per month.

Language: The Easiest Decision Variable

If you need anything other than English, use ElevenLabs.

I use it for Spanish. The multilingual model (eleven_multilingual_v2) sounds like a native speaker. The difference between that and using an English-trained model on Spanish text is significant: English-trained voices on Spanish sound like someone reading phonetically from a script. Flat, slightly wrong stress patterns, occasionally embarrassing pronunciations.

I learned this by trying it first. I used one of ElevenLabs' English voices on a Spanish lesson thinking it wouldn't matter that much. It mattered. I switched to Gaby (voice_id: n4GNpJP6Y2Nd09pDtetA) on the multilingual model and it sounded like an actual person.

Kokoro's English voices are good. Its multilingual support is improving but English is where it's been most tested. If your agent speaks English, you have a real choice to make. If it doesn't, ElevenLabs is the answer and you can stop reading here.

Cost: The Math Is Simple

ElevenLabs pricing as of 2026:
- Free tier: 10,000 characters/month
- Starter ($5/month): 30,000 characters/month
- Creator ($22/month): 100,000 characters/month

A typical voice note is 200-500 characters of input text. One sentence is roughly 60-100 characters.

At 3 Spanish lessons per day with ~300 characters each: 900 chars/day × 30 days = 27,000 chars/month. That's right at the $5 Starter tier.

At 10 English voice notes per day with ~400 characters each: 4,000 chars/day × 30 days = 120,000 chars/month. That's past the Creator tier ($22). That's when Kokoro starts making financial sense.

The crossover point where Kokoro becomes cheaper: roughly 3,300 characters per day, if you already have a GPU. If you don't have a GPU (or you're on a laptop), Kokoro still works but it's slow enough that ElevenLabs is worth paying for.

Kokoro's cost if you have an RTX 3090 or similar: basically zero. Electricity you're already paying for. GPU time on a card that would otherwise be idle.

The GPU Question

Kokoro runs on CPU. It also runs on CUDA (NVIDIA GPU) and MPS (Apple Silicon). CPU is slow, though: for a 200-word voice note, you're looking at 20-40 seconds on a decent CPU vs 2-3 seconds on a GPU.

If you're running cron jobs, you probably don't care about 40 seconds. The cron runs at 6am and delivers to your phone by 6:00:40. Fine.

But if you're doing this frequently or in a context where latency matters, you want GPU. I run Kokoro on my RTX 3090. Generation is fast enough that I don't think about it.

The install is real work. You need CUDA set up properly, Python dependencies in order, and the model downloaded. My first run failed because of a mismatch between my CUDA version and a dependency. The second attempt worked.

One thing I'd warn you about: Kokoro doesn't auto-start with my machine. I start it when I need it (cd ~/workspace/kokoro-tts && uv run python serve.py). My cron jobs that use Kokoro check the health endpoint first and fall back to ElevenLabs if Kokoro isn't running. That fallback saved me several times in the first month.

The Hybrid Approach

I use both. ElevenLabs for Spanish. Kokoro for English.

The routing logic is straightforward: detect what language the text is in, call the right provider. If Kokoro is down, fall back to ElevenLabs. The code bundle in the Voice AI Starter Kit has a tts_router.py that handles this.

The reason this setup makes sense: Spanish audio is low volume (a few lessons per day), high quality requirement (I'm learning the language, accents matter), and multilingual capability is required. ElevenLabs at $5/month is fine for that.

English audio is higher volume (multiple crons, daily briefings), English quality from Kokoro is good, and I have the GPU to run it. Kokoro at zero marginal cost makes sense.

If you're starting out and don't want to manage two providers, ElevenLabs-only is a completely reasonable setup. The $5/month tier covers most personal agent workflows. Add Kokoro later if you hit the cost ceiling.

Delivery: One Gotcha You'll Run Into

When you send audio to Telegram, there's a difference between sendAudio and sendVoice.

sendAudio delivers as a music file. The player shows a waveform with forward/back controls, treats it like a song.

sendVoice delivers as a voice message. The circular play button, the way a voice memo looks. It's the right format for agent audio briefings.

Kokoro outputs WAV. Telegram's sendVoice requires Opus (OGG container). You need an ffmpeg conversion step in between:

ffmpeg -y -i input.wav -c:a libopus output.opus

Then send the .opus file with sendVoice. If you skip this and send the WAV directly, Telegram will accept it as sendAudio and it'll look like a music file.

ElevenLabs can output MP3. Same issue: you need ffmpeg to convert to Opus before sendVoice.

The telegram_voice.py file in the code bundle handles this pipeline: audio bytes in, Opus voice note in Telegram out.

The Short Version

For most people starting out:

  • Non-English language needed: ElevenLabs, multilingual model, find a native voice in their library. $5/month.
  • English only, have a GPU, high volume: Kokoro. One-time setup effort, then free.
  • English only, no GPU, low volume: ElevenLabs Starter tier. $5/month, don't bother with Kokoro.
  • Want both languages and high volume: Hybrid routing. ElevenLabs for non-English, Kokoro for English.

Voice output is one of those agent features that sounds more complicated than it is. Once it's wired in, it runs and you forget about it. The setup is the hard part.
