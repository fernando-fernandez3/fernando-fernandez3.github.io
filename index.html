FAQ

Questions

The setup is documented using Paperclip for agent orchestration, but the architecture principles apply to any system. Claude Code, OpenAI Assistants, custom scripts, or bare cron all work. The cron configs and memory system work with whatever orchestrator you're running. The guide explains the reasoning so you can adapt it, not just copy it.
You should be comfortable with a terminal and basic file operations. This is a technical setup for people who want to actually understand and own their agent system. If you've run a cron job or set up a Telegram bot before, you'll be fine. If you've never touched a terminal, start somewhere simpler.
The guide is built around the Claude API (Anthropic), which is what the documented system runs on. The memory architecture and cron patterns are model-agnostic. Monitoring setup too. The token optimization chapter is Claude-specific (Haiku / Sonnet / Opus routing), but the principles apply to any tiered model pricing. Chapter 5 on TTS covers both ElevenLabs and Kokoro.
My setup runs at roughly $2.50–$3.50/day with 10 cron jobs and regular interactive use. That's ~$75–105/month. The token optimization chapter breaks down exactly where the spend goes and how to reduce it. The included token dashboard gives you live visibility. Most of the cron jobs use Haiku and cost less than $0.02/run.
AI agent tooling moves fast. The guide is free and will be updated as the architecture evolves. If you bought the bundle, config file updates are included too. The core architecture (memory, scheduling, monitoring) is more durable than any specific tool walkthrough.
